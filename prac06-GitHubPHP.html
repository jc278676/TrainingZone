PHP (CP2010 + keen CP1406 students):

Please download and unzip the sample files provided (more on these later).

Remember that PHP is server-side, so it doesn't run in the browser (like HTML/CSS), but on the server. You need to upload and view on the server, or a localhost server (like USBWebServer).

Basics:

  1. Create a new directory on your server under public_html for this prac. You could call it php or something.
  2. Create a 'blank' ("None" for layout) PHP file (choose HTML5 docType) in Dreamweaver (or your choice of program). You may notice that the ONLY difference between this PHP file and a blank HTML file is that when you save it, the default file name ends in .php.
    Remember that every file you write that you want to use PHP code in must have a .php extension.
    Add a simple PHP <?php echo "Hello World"; ?> statement in the body somewhere. Upload it to your server and test it.
    Change it (add another statement), re-upload and refresh the page - to make sure your workflow is working and flowing...
  3. Create a "template" page where you include a header file and a footer file.
    Make the header and footer files contain valid content, like a site title in the header and a copyright notice in the footer.

Databases:

  1. Download the phonerecords.zip file from here on LearnJCU.
    Unzip it and upload the contents to a directory called phonerecords on your server (you don't need the MACOSX files or the _notes files, just the php and the sqlite files).
  2. Point your Web browser to the new files - something like http://ditwebtsv.jcu.edu.au/~jc123456/phonerecords
    "Play" with the simple phone records example and see what it can do. Start with the phone.php file.
  3. To begin with, it will not insert or modify data - this is a permissions issue.
    You need to make the database file and the directory it's in writeable by other (everyone).
    You can do this by changing to "Remote server" view in the Files panel in Dreamweaver, right-clicking on these two (file and directory) and choose Set Permissions...They should be:
    • database file:
    • directory database is in:
  4. Spend some time reading the source code in Dreamweaver so you can see how this demo works.
    The example is not meant to demonstrate beautiful design, but it is a fairly complete demo of just about all the things you need to do with your database programming, including:
    1. Insert a new record
    2. Update an existing record (make changes to the text in the fields) then click Update Entry
    3. Delete existing records. In this demo you can do this either by
      1. clicking the Delete Entry button (takes you to the output page), or
      2. clicking the X button ("silently" deletes the record and redirects you back to the main page using the PHP header function)
  5. Download Adminer from http://www.adminer.org
    Rename the file to just adminer.php then upload it to your public_html folder
  6. Load this in your browser and enter the path to your phonerecords db file.
    Then 'play' with this to see how it works.
    Make changes to the database using Adminer then check the changes on the phonerecords site.
  7. Good, now you need to do some of your own additions to this program.
    Let's start by simply adding heading text to the list under "Current Data". We see the values, but no headings for these, so add "Name" and "Phone" above the listings. Don't worry about the lining up of this at this stage. You can come back and use CSS to display this properly some time if you like.
  8. Now display the record ID as normal text (not in a form field) at the start of each line of output. See if you can figure out where this should go.
  9. Let's add another field to the database.
    The current example only has name and phone fields. Add a new (text) field for address to the database using SQLite Manager.
  10. Modify the PHP code so that it can handle the new address field.
    Just look at how the existing code works and update it based on what you see there.
  11. Good... Now start from scratch and make your own complete but simple database-driven Web site.
    Please do this iteratively. First get a small part working - like displaying database content - then add functionality one step at a time, testing along the way.
    Let's make a simple site to display and work with products for sale. You can customise this and add more if you like.
    Start by making a new database file in SQLite Manager. Include the following fields:
    1. id (auto increment integer)
    2. name
    3. description
    4. price
  12. Create PHP code to:
    1. display the contents of the database
    2. delete records
    3. update records
    4. (advanced - not in sample code) ... create tick boxes or quantity fields in HTML next to each product so you can choose items to buy, then a BUY button that will show the total price when clicked.
